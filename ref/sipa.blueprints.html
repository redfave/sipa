sipa.blueprints.generic.exceptionhandler_ldap(ex)[source]¶

Handles global LDAPCommunicationError exceptions.

The session must be reset, because if the user is logged in and the server fails during his session, it would cause a redirect loop. This also resets the language choice, btw.

The alternative would be a try-except catch block in load_user, but login also needs a handler.
