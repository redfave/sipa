sipa.config package¶

Submodules¶

sipa.config.default module¶

all configuration options and dicts of external information (dormitory mapping etc.) Project-specific options should be included in the config.py, which is a file not tracked in git containing IPs, user names, passwords, etc.

sipa.config.example module¶

config.example

This is an example configuration file for SIPA, suggesting defaults for deployment.

Every value not commented with “Must be set” has been given the default value which is assigned in the comment.
