class sipa.model.fancy_property.active_prop(fget, fset=None, fdel=None, doc=None, fake_setter=False)[source]¶

Bases: property

A property-like class wrapping the getter with ActiveProperty

active_prop automatically adds edit and delete capabilities to the ActiveProperty object if setter/deleter is invoked.

Example usage:

>>> class User:
...     @active_prop
...     def foo(self):
...         return {'value': "Empty!!", 'empty': True, 'style': 'danger'}
...
...     @active_prop
...     def bar(self):
...         return 0
...
...     @bar.setter
...     def bar(self):
...         print("furiously setting things")
>>> User().foo
<ActiveProperty foo='Empty!!' [empty]>
>>> User().bar
<ActiveProperty bar='Nicht angegeben' [empty]>
>>> User().bar.capabilities
capabilities(edit=True, delete=False)

sipa.model.finance module¶

class sipa.model.finance.BaseFinanceInformation[source]¶

Bases: object

A Class providing finance information about a user.

This class bundles informations such as whether the current user is obliged to pay, the current balance, history, and the time of the last update. The balance is being provided as a FancyProperty, so it can be used as a row.

For subclassing, implement the respective abstract methods/properties.

balance¶

The current balance as a ActiveProperty

If has_to_pay is False, return an empty property with the note that the user doesn’t have to pay. Else, return the balance formatted and styled (red/green) as money and mark this property editable.

[Abstract] Whether the user is obliged to pay.

[Abstract] History of payments

This method should return an iterable of a (datetime, int) tuple.

[Abstract] The time of the last update.

raw_balance¶

[Abstract] The current balance

If has_to_pay() is False, this method will not be used implicitly.

class sipa.model.user.BaseUser(uid)[source]¶

Bases: sipa.model.user.AuthenticatedUserMixin

Abstract base class defining what a user must have in order to work properly with sipa.

Note that initialization shouldn’t be done via __init__() directly, but usign one of the classmethods get(), authenticate() or from_ip()!

Apart from being an ABC, this class provides some logic already.

This includes setting uid in __init__(), defining equality, and some others.

This class adheres to the protocol imposed by Flask-Login

tmp_authentication(password)[source]¶

Check and temporarily store the given password.

Returns a context manager. The password is stored in self._tmp_password.

This is quite an ugly hack, only existing because some datasources need the user password to change certain user properties such as mail address and MAC address. The need for the password breaks compatability with the usual instance.property = value.

I could not think of a better way to get around this.

traffic_history¶

Return the current traffic history as a dict.

The history should cover one week. The assumed unit is KiB.

The list syntax is as follows:

[{
    'day': day.weekday(),
    'input': in,
    'output': out,
    'throughput': in + out,
}, …]

The traffic values shall be in KiB, as usual.

Returns:The history of the used traffic
